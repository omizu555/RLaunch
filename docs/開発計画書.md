## 1. プロジェクト概要

CLaunch に着想を得た、ボタン型アプリケーションランチャーを React + Tauri v2 で新規開発する。  
デスクトップに常駐し、ホットキーやマウス操作で呼び出せるグリッド形式のランチャーウィンドウを提供する。  
Stream Deck を参考に、ボタン上にアナログ時計・デジタル時計・システムモニター等の動的ウィジェットを表示する機能も備える。  
ウィジェットとテーマはプラグイン方式により、ユーザーが外部フォルダに追加して拡張可能。

## 6. 技術的な注意点

### 6.1 Tauri v2 のパーミッションモデル
全てのプラグイン API は `capabilities/default.json` に明示的に許可を記述。  
現在は `main`, `settings`, `widget-settings`, `widget-select` の4ウィンドウに権限付与。

### 6.2 フレームレスウィンドウ
- `decorations: false` でカスタムタイトルバーを自前実装
- `data-tauri-drag-region` でドラッグ移動
- ドラッグ時のフォーカス喪失を `isWindowDragging` フラグで回避

### 6.3 マルチウィンドウ構成
各サブウィンドウは Tauri `emit/listen` で通信。Vite マルチページビルドで個別エントリポイントを持ち `WebviewWindow` で動的生成。  
`useChildWindow<TInit, TResult>` ジェネリックフックにより、ready → init ハンドシェイク・ウィンドウ再利用・extra listeners 拡張を共通化。  
現在 7 ウィンドウ: main, settings, widget-settings, widget-select, folder-browser, group-edit, group-popup。

### 6.4 セル間ドラッグ&ドロップ
ポインタイベントベース（`pointermove` + `elementFromPoint`）で実装。  
HTML5 Drag API は Tauri ネイティブ D&D と競合するため使用しない。  
OS ファイルドロップは `useDragDrop` フック（Tauri `onDragDropEvent`）で別途処理。

### 6.5 ウィジェットプラグインのスクリプト実行
`new Function(script + "; return draw;")()` のファクトリパターンで JS を一度だけ実行し、  
draw 関数をクロージャとして取得・キャッシュ。グローバル状態はクロージャ内に閉じ込め。

### 6.6 テーマの動的適用
テーマは JSON（CSS変数 key-value）。`:root` に `setProperty` で注入し、`data-theme-vars` 属性で管理。

### 6.7 起動方式 (ShellExecuteW)
`launch_app` / `run_as_admin` は Win32 `ShellExecuteW` で直接実行。  
`Command::new()` による子プロセス起動を廃止し、CMD ウィンドウのフラッシュを完全回避。

### 6.8 サブグループ (GroupItem)
`GridCell` のひとつとして `GroupItem` を配置可能。  
クリックで独立ウィンドウのミニランチャーが開き、グループ内アイテムを起動/追加/削除できる。  
グループの変更は `group-popup-update` イベントでメインウィンドウに即時同期。

### 6.9 ファイル登録パイプライン
`fileRegistration.ts` の `createLauncherItemFromPath()` が統一的なアイテム生成を担当。  
`get_file_info` → `.lnk` 解決 (`resolve_lnk`) → アイコン抽出 (`extract_icon`) → `LauncherItem` 構築。  
メイングリッドと GroupPopupWindow の両方で共通使用。
