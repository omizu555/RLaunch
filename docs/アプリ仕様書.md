## 1. 機能概要

| # | 機能 | 説明 |

## 2. ウィンドウ構成

### 2.1 メインウィンドウ (`main`)

```
┌─────────────────────────────────────────────────┐
│ [☰][タイトル]              [📌][−][×]           │ ← タイトルバー (36px)
├─────────────────────────────────────────────────┤
│ [Tab1][Tab2][Tab3][+]                           │ ← タブバー (36px)
├─────────────────────────────────────────────────┤
│                                                 │

│  │  │ │  │ │  │ │  │ │  │ │  │ │  │ │  │    │ ← グリッド
│  └──┘ └──┘ └──┘ └──┘ └──┘ └──┘ └──┘ └──┘    │   (cellSize × cols × rows)
│  ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐    │

│  │  │ │  │ │  │ │  │ │  │ │  │ │  │ │  │    │

│                                                 │
```

- **フレームレス**: `decorations: false` でカスタムタイトルバー
- **サイズ自動計算**: `cellSize × cols + gap × (cols-1) + padding` で幅・高さを算出
- **表示位置**: `center` / `cursor` / `remember` から選択可能
- **最前面固定**: `alwaysOnTop: true`

### 2.2 設定ウィンドウ (`settings`)

独立 WebviewWindow。親ウィンドウのタイトルバー ☰ ボタンまたはトレイメニューから開く。

| セクション | 設定項目 |
|---|---|
| 一般 | ホットキー、表示位置、自動非表示、起動後非表示、自動起動 |
| グリッド | デフォルト列数(1〜20)、行数(1〜10)、セルサイズ(32〜128px) |
| ラベル | ボタンラベルの表示/非表示 |
| テーマ | テーマ選択（リアルタイムプレビュー対応） |
| エフェクト | ウィンドウエフェクト（none / mica / acrylic） |

設定変更は `emit("settings-changed")` でメインウィンドウへリアルタイム反映。  
ウィンドウ閉じ時に `emit("settings-closed")` を送信。

### 2.3 ウィジェット選択ウィンドウ (`widget-select`)

独立 WebviewWindow。セル右クリック→「ウィジェットの追加」で開く。

- 全ウィジェット（ビルトイン + プラグイン）をリスト表示
- 各ウィジェットのメタ情報（絵文字、名前、作者、説明）を表示
- 選択時に `emit("widget-select-result", { widgetId, index })` を送信

### 2.4 ウィジェット設定ウィンドウ (`widget-settings`)

独立 WebviewWindow。配置済みウィジェットの右クリック→「ウィジェットの設定」で開く。

- `configSchema` に基づく動的フォーム生成
- フィールドタイプ: `color`, `checkbox`, `select`, `text`, `number`, `datetime`
- 保存時に `emit("widget-settings-result", { index, config })` を送信

## 3. データ構造

### 3.1 AppSettings（アプリ全体設定）

```typescript
interface AppSettings {
  autoStart: boolean;           // OS起動時に自動実行
  hotkey: string;               // ホットキー ("Ctrl+Space")
  defaultGridColumns: number;   // デフォルト列数 (8)
  defaultGridRows: number;      // デフォルト行数 (4)
  cellSize: number;             // セルサイズ px (64)
  showLabels: boolean;          // ボタンラベル表示
  viewMode: "grid" | "list";    // 表示モード ("grid") - 全体デフォルト
  listColumns: number;          // リスト列数 (1-4) - 全体デフォルト
  theme: string;                // テーマID ("dark")
  windowEffect: "none" | "mica" | "acrylic";
  autoHide: boolean;            // フォーカス喪失で非表示
  hideOnLaunch: boolean;        // アプリ起動後に非表示
  windowPosition: "center" | "cursor" | "remember";
  windowX?: number;             // remember 時のX座標
  windowY?: number;             // remember 時のY座標
}
```

### 3.2 Tab（タブ）

```typescript
interface Tab {
  id: string;
  label: string;
  order: number;
  gridColumns: number;
  gridRows: number;
  items: GridCell[];            // 長さ = columns × rows
  viewMode?: "grid" | "list";  // タブ個別表示モード (未設定時は全体設定を使用)
  listColumns?: number;        // タブ個別リスト列数 (未設定時は全体設定を使用)
}
```

### 3.3 GridCell（グリッドセル）

```typescript
type GridCell = LauncherItem | WidgetItem | GroupItem | null;
```

### 3.4 LauncherItem（登録アプリ）

```typescript
interface LauncherItem {
  id: string;                   // UUID
  label: string;
  path: string;                 // 実行ファイルパス
  args?: string;                // 起動引数
  workingDir?: string;          // 作業ディレクトリ
  iconBase64?: string;          // 抽出アイコン (Base64)
  iconPath?: string;            // アイコンファイルパス
  type: ItemType;               // "executable" | "shortcut" | "folder" | "url" | "document" | "widget"
  runAs?: boolean;              // 管理者として実行
  windowState?: WindowState;    // "normal" | "maximized" | "minimized"
  hotkey?: string;
  createdAt: string;            // ISO 8601
  updatedAt: string;
}
```

### 3.5 WidgetItem（ウィジェット）

```typescript
interface WidgetItem {
  id: string;
  type: "widget";
  widgetType: WidgetType;       // string（ビルトイン or プラグインID）
  label?: string;
  config: Record<string, unknown>;
  updateInterval: number;       // ms
  createdAt: string;
  updatedAt: string;

```

### 3.6 GroupItem（サブグループ）

```typescript
interface GroupItem {
  id: string;
  type: "group";
  label: string;
  icon?: string;                // カスタム絵文字アイコン
```

## 4. テーマシステム

### 4.1 テーマ構造

テーマは JSON 形式で CSS 変数の key-value を定義。

```json
{
  "id": "dark",
  "name": "Dark",
  "colors": {
    "--bg-primary": "#1e1e2e",
    "--bg-secondary": "#313244",
    "--text-primary": "#cdd6f4",
    "--accent": "#89b4fa",
    ...
  }
}
```

### 4.2 読み込みフロー

1. アプリ起動時: Rust `init_themes` → ビルトイン JSON をAppDataにコピー
2. フロントエンド: `listThemes()` → Rust `list_themes` → 全テーマ一覧
3. テーマ適用: `applyThemeById()` → JSON読み込み → `:root` に `setProperty`
4. 前回値クリア: `data-theme-vars` 属性で追跡し、テーマ切替時にリセット

## 7. 操作仕様

### 7.1 アプリ起動フロー

1. ダブルクリック / ホットキー でメインウィンドウ表示
2. ブラウザデフォルト右クリックメニューは `contextmenu` イベントで抑制

### 7.2 セル操作（左クリック）

| セル状態 | 動作 |
|---|---|
| アプリ登録済み | アプリを起動 |
| ウィジェット配置済み | ウィジェットのクリックハンドラを実行 |
| 空セル | 何もしない |

### 7.3 セル操作（右クリック → コンテキストメニュー）

| メニュー項目 | 条件 | 動作 |
|---|---|---|
| アプリの起動 | アプリ登録済み | 通常起動 |
| 管理者として実行 | アプリ登録済み | 特権昇格起動 |
| ファイルの場所を開く | アプリ登録済み | エクスプローラーで親フォルダ |
| 削除 | アイテムあり | セルをクリア |
| ウィジェットの追加 | 空セル | ウィジェット選択ウィンドウを開く |
| ウィジェットの設定 | ウィジェット配置済み | ウィジェット設定ウィンドウを開く |

### 7.4 D&D 操作

#### OS ファイルドロップ（外部 → グリッド）
- Tauri ネイティブ `onDragDropEvent()` で検知
- `.lnk` → Rust でリンク先解決 → 登録
- `.exe` / フォルダ → 直接登録
- ドロップ座標から `data-cell-index` でセル特定
- ドラッグオーバー時にハイライト表示

#### セル間移動（内部グリッド内）
- ポインタイベントベース（`pointerdown` → `pointermove` → `pointerup`）
- `DRAG_THRESHOLD = 5px` でクリックと区別
- `elementFromPoint()` でドロップ先セルを特定
- ソースアイテム: `opacity: 0.4, scale: 0.92, dashed border`
- ターゲットセル: `solid accent border, scale: 1.05`
- HTML5 Drag API は不使用（Tauri ネイティブ D&D と競合するため）

### 7.5 タブ操作

| 操作 | 動作 |
|---|---|
| タブクリック | タブ切替 |
| ＋ ボタン | 新規タブ追加 |
| ダブルクリック | タブ名変更 |
| × ボタン | タブ削除（確認なし） |
| マウスホイール | 前後タブ切替 |

### 7.6 検索

- `Ctrl+F` で検索バー表示
- 全タブの登録アイテムからインクリメンタルサーチ
- 結果クリックで対象タブに切替＋アプリ起動

### 7.7 ホットキー

| ホットキー | 動作 |
|---|---|
| `Ctrl+Space` (デフォルト) | メインウィンドウ表示/非表示 |
| `Ctrl+F` | 検索バー開閉 |

## 8. ウィンドウサイズ自動計算

ウィンドウサイズは `useWindowSize` フックで以下の式に基づき算出:

```
width  = cellSize × cols + GRID_GAP × (cols - 1) + GRID_PADDING + BORDER_EXTRA
height = cellSize × rows + GRID_GAP × (rows - 1) + GRID_PADDING + BORDER_EXTRA
         + TITLEBAR_HEIGHT + TABBAR_HEIGHT + STATUSBAR_HEIGHT
```

レイアウト定数:

| 定数 | 値 (px) | 説明 |
|---|---|---|
| GRID_GAP | 6 | セル間ギャップ |
| GRID_PADDING | 20 | グリッドパディング (左10+右10) |
| BORDER_EXTRA | 2 | ボーダー余白 |
| TITLEBAR_HEIGHT | 36 | タイトルバー |
| TABBAR_HEIGHT | 36 | タブバー |
| STATUSBAR_HEIGHT | 28 | ステータスバー |

### グリッドモード (`viewMode: "grid"`)

上記の計算式で算出。

### リストモード (`viewMode: "list"`)

```
COMPACT_ROW_HEIGHT = 32  // --compact-row-height
COMPACT_GAP = 2
listColumns = タブ個別設定 ?? 全体設定 ?? 1
compactRows = ceil(cols × rows / listColumns)
width  = cellSize × cols + GRID_GAP × (cols - 1) + GRID_PADDING + BORDER_EXTRA
height = COMPACT_ROW_HEIGHT × compactRows + COMPACT_GAP × (compactRows - 1) + GRID_PADDING
         + TITLEBAR_HEIGHT + TABBAR_HEIGHT + STATUSBAR_HEIGHT + BORDER_EXTRA
```

リストモードはグリッドモードと同じレイアウトエンジン（LauncherGrid）を使用し、
セルが横長の長方形になるコンパクト表示です。空セルも表示され、ウィジェットも動作します。
ドラッグ＆ドロップ、コンテキストメニュー、キーボードナビゲーション等の
全機能がグリッドモードと共通で利用可能です。

## 9. ウィンドウ間通信

各ウィンドウは Tauri `emit/listen` イベントで通信。

### 9.1 設定ウィンドウ

```
main → settings:  "settings-current"   { settings: AppSettings }
settings → main:  "settings-changed"   { settings: AppSettings }
settings → main:  "settings-closed"    {}
```

### 9.2 ウィジェット選択ウィンドウ

```
main → widget-select:  "widget-select-init"    { index: number }
widget-select → main:  "widget-select-result"  { widgetId: string, index: number }
widget-select → main:  "widget-select-closed"  {}
```

### 9.3 ウィジェット設定ウィンドウ

```
main → widget-settings: "widget-settings-init"   { index, widgetType, config, manifest }
widget-settings → main: "widget-settings-result"  { index, config }
widget-settings → main: "widget-settings-closed"  {}
```

### 9.4 グループポップアップウィンドウ

```
main → group-popup:    "group-popup-init"    { group: GroupItem, parentViewMode, parentListColumns }
group-popup → main:    "group-popup-update"  { group: GroupItem }
group-popup → main:    "group-popup-launch"  { item: LauncherItem }
group-popup → main:    "group-popup-closed"  {}
group-popup → main:    "group-popup-action"  { action: "browse-folder", cellIndex, path }
```

グループポップアップはファイル/URL登録・起動・編集のコンテキストメニューを提供。
ウィジェット・フォルダ選択はサポート外。フォルダ参照のみ親に委譲します。
