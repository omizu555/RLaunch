## 1. 設計原則 — Fitts の法則とマウス操作

本提案書の全アイデアは、HCI（ヒューマン・コンピュータ・インタラクション）の基本法則に基づいています。

### Fitts の法則

$$T = a + b \cdot \log_2\left(\frac{D}{W} + 1\right)$$

- $T$: ターゲットに到達する時間
- $D$: カーソルからターゲットまでの距離
- $W$: ターゲットの幅（大きさ）

**要約: ターゲットは大きく近いほど速く操作できる。**

この法則から導かれる設計原則:

| 原則 | 応用 |
|---|---|
| 画面端は「無限に大きい」 | 端にスナップする操作は超高速（ホットエッジ） |
| 円形メニューは均等距離 | 全選択肢への到達時間が等しい（パイメニュー） |
| 近くの大きなターゲット最速 | ホバー先に拡大表示（マグニフィケーション） |
| 動かす距離を減らす | カーソル位置に UI を出す（右クリックメニュー改良） |

## 2. パイメニュー（円形メニュー）

### 着想元
- **Blender**: 3D ビューの操作でパイメニューが標準装備。8方向の選択が瞬時にできる
- **The Sims**: アイテムとのインタラクション時に円形メニューが出現
- **GTA V**: 武器ホイール。ゲームパッドでもマウスでも直感的

### 現状の問題
右クリックで表示される縦型コンテキストメニューは、項目数が増えると下方向にマウスを大きく動かす必要がある。Fitts の法則的に非効率。

### 提案: M-01 セルの右クリックをパイメニューに置換

```
       ▶ 起動
        ╱
 📂開く ─── ● ─── ✏ 編集
        ╲
       🗑 削除
```

- 右クリック位置を中心に半径 60〜80px の円形メニューを表示
- 各項目を等間隔に配置（4 項目なら 90° 間隔、6 項目なら 60° 間隔）
- カーソルを方向に動かすだけで選択 → **全項目が同じ距離** → Fitts の法則で最速
- 方向にマウスを動かすと該当セクションがハイライト
- マウスボタンを離すと実行（Press-Move-Release パターン）

**実装アイデア:**
```tsx
// カーソル角度から選択中の項目を算出
const angle = Math.atan2(dy, dx) * (180 / Math.PI);
const sector = Math.round(((angle + 360) % 360) / sectorAngle) % itemCount;
```

**切替可能:** 設定で「リニアメニュー / パイメニュー」を選択可能に。

## 3. マウスジェスチャー

### 着想元
- **Opera ブラウザ**: 右クリック + 方向でページ操作。何百万人が常用した実績あり
- **Vivaldi**: カスタマイズ可能なマウスジェスチャー
- **StrokeIt / StrokesPlus**: Windows 用マウスジェスチャーソフト

### 提案: M-03 グリッド上のマウスジェスチャー

右ボタンを押しながらマウスを特定方向に動かすと操作を実行。

| ジェスチャー | 動作 | 覚え方 |
|---|---|---|
| → (右) | 次のタブに切替 | ページめくり |
| ← (左) | 前のタブに切替 | ページ戻し |
| ↑ (上) | 検索バーを開く | 上に探す |
| ↓ (下) | 設定を開く | 下に潜る |
| →← (右左) | 最後に使ったアプリを再起動 | 往復 = リピート |
| ↑↓ (上下) | ピン留めトグル | 上下 = 切替 |
| L字 (↓→) | 新しいタブを追加 | L = 新たな空間 |
| 逆L字 (→↓) | フォーカスタブを閉じる | × を描く始まり |

**ジェスチャー認識:**
```
1. 右ボタン押下を検知
2. 閾値(20px)以上の移動で方向を記録（上/下/左/右）
3. 方向転換を検知してシーケンスに追加
4. 右ボタンリリースでシーケンスを評価
5. 一致するジェスチャーがあれば実行、なければ通常の右クリックメニュー表示
```

**視覚フィードバック:**
- ジェスチャー軌跡をキャンバス上に半透明の線で描画（操作中のみ）
- 認識完了時にジェスチャー名を一瞬表示（例: 「→ 次のタブ」）

---

### 提案: M-04 セル上のコンパクトジェスチャー

セルを右クリックした後、**離さずに方向へドラッグ** して操作を選択。

- 上にドラッグ → 管理者として起動
- 右にドラッグ → ファイルの場所を開く
- 下にドラッグ → 編集
- 離した位置が中心近く → 通常メニュー表示（従来のフォールバック）

パイメニューのさらに軽量版。視覚表示なしでも機能する「見えないパイメニュー」。
上級者はメニューを見ずに操作できる。

## 4. ホットエッジ＆コーナートリガー

### 着想元
- **Windows 8**: 画面角ホバーでチャームバー出現
- **macOS**: 画面コーナーにMission Control/デスクトップ表示を割当

### 提案: M-05 ランチャーウィンドウの端をアクション領域に

ランチャーウィンドウの四辺・四隅に、マウスカーソルが到達したときに特定アクションを発動。

```
┌─[検索]──────────────────────────[設定]─┐
│                                         │
[←Tab]          グリッド           [Tab→] │
│                                         │
└─[新規登録]──────────────────[最近使った]─┘
```

| エッジ/コーナー | アクション |
|---|---|
| 上辺 | 検索バーをスライドイン |
| 左辺 | 前タブへ切替（ホイール不要） |
| 右辺 | 次タブへ切替 |
| 下辺 | ステータスバー詳細表示 |
| 左上角 | 検索（Fitts 最大化: 角は無限ターゲット） |
| 右上角 | 設定ウィンドウを開く |
| 左下角 | ファイルダイアログで新規登録 |
| 右下角 | 最近起動したアイテムをポップアップ |

**ポイント: 画面端は Fitts の法則で「無限に大きい」ターゲット。** カーソルが壁にぶつかって止まるため、オーバーシュートが起きない。ランチャーウィンドウ端にもこれを応用。

### 提案: M-06 デスクトップホットコーナーでランチャー呼び出し

`Ctrl+Space` のホットキーに加え、**画面端カーソル接触**でもランチャーを呼び出し可能にする。

- 例: マウスカーソルを画面左下に 300ms 滞留 → ランチャーがフェードイン
- 割り当てる角を設定で選択
- 滞留時間も変更可能（200ms〜1000ms）
- Tauri の Rust 側でグローバルなカーソル位置を定期ポーリング（`get_cursor_position` は既存）

## 5. Dock 型マグニフィケーション

### 着想元
- **macOS Dock**: カーソルに近いアイコンが拡大し、隣接アイコンも段階的に拡大
- **Plank / Cairo Dock (Linux)**: 同様のマグニフィケーション効果

### 提案: M-07 グリッドセルのポインタ追従拡大

カーソルの位置に応じてセルを連続的にスケーリングする。

```
通常:     [48] [48] [48] [48] [48]

ホバー:   [48] [56] [72] [56] [48]
                     ↑ カーソル位置
```

- ホバーされたセル: `scale(1.15)`
- 隣接セル: `scale(1.07)`
- 2つ隣のセル: `scale(1.02)`
- ガウス分布に基づいてスケーリング量を算出

**計算式:**
```tsx
// ホバーされたセルからの距離に基づくスケール
const distance = Math.abs(cellIndex - hoveredIndex);
const scale = 1 + 0.15 * Math.exp(-0.5 * (distance / sigma) ** 2);
// sigma ≈ 1.2 で macOS Dock に近い挙動
```

- CSS `transform: scale()` + `transform-origin` で実装
- `will-change: transform` で GPU アクセラレーション
- 設定で ON/OFF 切替可能（好みが分かれる機能のため）

## 6. スプリングローデッドフォルダ

### 着想元
- **macOS Finder**: ファイルをフォルダ上でホールドすると、フォルダが自動的にスプリングのように「パカッ」と開く
- **Windows Explorer**: 同様のSpring-loaded機能（ただし存在感薄い）

### 提案: M-08 D&D 中のグループ自動展開

アイテムをドラッグ中にグループセルの上で一定時間(500ms)ホールドすると:

1. グループポップアップが自動的に開く
2. そのままポップアップ内のセルにドロップ可能
3. ポップアップから外にドラッグすると自動的に閉じる

```
[ドラッグ開始] → [グループ上で500msホールド] → [ポップアップ展開] → [グループ内にドロップ]
```

**これにより、メイン→グループ間のアイテム移動が D&D 一発で完了する。**

---

### 提案: M-09 D&D 中のタブ自動切替

アイテムをドラッグ中にタブバー上でホールドすると:

1. 300ms 後にそのタブに切り替わる
2. 新しいタブのグリッドにドロップ可能
3. タブとグリッドをまたぐ D&D 操作が完了

**macOS Finder + Chrome タブの組合せ着想。** Chromeではタブ上へのファイルドラッグでタブが切り替わる。

### 提案: M-16 フォルダのホバー展開（ミニブラウザ）

フォルダ型アイテムの上に 800ms 滞留すると:

- セルの隣に **ミニファイルリスト（最大 10 件）** がドロップダウン表示
- リスト内の項目をクリックで直接起動
- リストの下に「📂 すべて表示」リンク → フルブラウザウィンドウを開く

**Amazon のメガメニューのように、カーソルの移動経路を考慮した三角形ヒットエリアで誤爆を防止:**

```
  カーソル移動方向 →

     ╲[項目2       ]
```
カーソルがメニューに向かって斜めに動いている間は、途中のセルに触れてもメニューを閉じない。

## 10. ゾーン分割セル

### 着想元

### 提案: M-17 1セル内を機能ゾーンに分割

セルの内部を4つのゾーンに分割し、クリック位置で異なるアクション:

```
┌──────────────┐
│ ⚙  │    │ ×  │  ← 上部: 左=設定  右=登録解除
│─── │    │────│
│    │ 🌐 │    │  ← 中央: メインアクション（起動）
│    │    │    │
│─── │    │────│
│ 📂 │    │ ✏  │  ← 下部: 左=場所  右=編集
└──────────────┘
```

- **通常時**: ゾーンは見えない（アイコンのみ表示）
- **ホバー時**: 薄いガイドラインがフェードインして操作可能領域を示唆
- **中央 60% エリア**: 左クリック = 通常起動（最も大きいターゲット = 最速）
- **四隅の小さなゾーン**: 補助操作
- セルサイズが 80px 以上の場合のみ有効（小さいセルでは無効化）

**自動車のメーターと同じ発想: 限られたスペースに複数の情報/操作を「ゾーン」で共存させる。**

## 12. ドラッグスロー（慣性＆物理演算）

### 着想元
- **iOS / macOS**: スクロールの慣性（フリックしてから減速して止まる）
- **ピンボール**: ボールを弾く力で結果が変わる
- **Google Maps**: パン操作の慣性

### 提案: M-19 アイテムの「投げ渡し」

セル間 D&D の際に、マウスリリース時の**速度**に応じて挙動を変える:

| リリース速度 | 挙動 |
|---|---|
| 静止（通常のドロップ） | 現在のドロップ先にスワップ（既存） |
| 低速フリック | ①ドロップ先候補をハイライトして確認後確定 |
| 高速フリック（方向へ投げる） | ドラッグ方向の最初の空きセルまでスライドして配置 |

- 「投げる」という物理的な行為をマウス操作に応用
- リリース時のカーソル速度を計算（直前 50ms のΔposition / Δtime）
- 高速フリック時はアニメーション付き（アイコンがスライドして空きセルに着地）

## 15. カーソル近接リアクション

### 着想元
- **エレベーターボタン**: 押すと光る（即座のフィードバック）
- **ゲーム UI**: ホバーでボタンが「呼吸する」アニメーション
- **Web デザイン**: インタラクティブなカード UI（tilt.js）

### 提案: M-23 生きているグリッド

カーソルとセルの距離に応じて、セルが微細にリアクションする。

**レベル 1: グロー効果**
- ホバー中のセルがテーマカラーで柔らかく発光
- CSS `box-shadow` の `spread` と `blur` をカーソル距離で動的変更

**レベル 2: チルト（傾き）効果**
- セルがカーソル方向に微小に傾く（3D transform）
- `perspective: 800px; rotateX(2deg); rotateY(-3deg);`
- マウス位置からセル中心への角度で計算

**レベル 3: 波紋効果**
- カーソル移動に応じてグリッド全体に波紋アニメーション
- 水面に石を投げたような CSS animation

```css
.btn {
  transition: transform 0.15s ease, box-shadow 0.2s ease;
}
.btn[data-proximity="near"] {
  box-shadow: 0 0 12px 2px var(--accent-color);
  transform: perspective(800px) rotateY(var(--tilt-x)) rotateX(var(--tilt-y));
}
```

**エレベーターボタン的な「押す前から反応している」感覚で、操作への期待感を醸成。**

## 17. ダブルクリック・トリプルクリック拡張

### 着想元
- **テキストエディタ**: ダブルクリック=単語選択、トリプルクリック=行選択
- **macOS Finder**: ダブルクリック=開く、クイック操作別動作
- **ゲーム**: 連打で強攻撃

### 提案: M-25 クリック回数で操作を階層化

| クリック回数 | 操作 |
|---|---|
| シングルクリック | 通常起動（既存） |
| ダブルクリック | 管理者として起動 |
| トリプルクリック | ファイルの場所を開く |

- ダブルクリック判定: 250ms 以内の 2 回クリック
- シングルクリック確定までの遅延を最小化するため、1回目クリックで即座に起動し、2 回目で追加操作を行う方式
- または: 1 回目を 200ms 遅延して判定（クリック回数を確定してからアクション）
- 設定で遅延方式を選択可能

## 21. 優先度 & 実装指針

### 実装容易性×インパクト マトリクス

```
インパクト(高)
    │
    │  M-12[中クリック]     M-01[パイメニュー]
    │  M-21[お気に入り]     M-08[スプリングローデッド]
    │  M-15[ホバープレビュー] M-03[マウスジェスチャー]
    │  M-09[タブ自動切替]
    │  M-25[ダブルクリック]
    │  ─────────────────────────────────────
    │  M-10[ホイール拡張]    M-07[Dockマグニフィケーション]
    │  M-26[ロングプレス]    M-17[ゾーン分割]
    │  M-20[ゴミ箱ゾーン]    M-22[ゴムバンド選択]
    │  M-06[ホットコーナー]  M-18[フィッシュアイ]
    │
インパクト(低)
    └──────────────────────────────────────→
      実装容易(左)                 実装困難(右)
```

### 推奨 Phase 構成

| Phase | 提案 | テーマ | 工数目安 |
|---|---|---|---|
| **Phase I** | M-12, M-25, M-10 | マウスボタン活用（既存イベントの拡張のみ） | 小 |
| **Phase II** | M-15, M-09, M-08 | ホバー＆スプリングローデッド（D&D 中の自動展開） | 中 |
| **Phase III** | M-01, M-26 | パイメニュー＆アクションリング（新 UI コンポーネント） | 中 |
| **Phase IV** | M-03, M-04 | マウスジェスチャー（ジェスチャー認識エンジン） | 中 |
| **Phase V** | M-21, M-20 | お気に入りドック＆ゴミ箱ゾーン（レイアウト変更） | 中 |
| **Phase VI** | M-07, M-23 | 視覚効果（マグニフィケーション＆近接リアクション） | 中〜大 |
| **Phase VII** | M-22, M-24 | 範囲選択＆方角ドラッグ（高度なポインタ操作） | 大 |
| **Phase VIII** | M-18, M-29, M-30 | 実験的 UI（フィッシュアイ＆カルーセル） | 大 |

### 重要な設計方針

1. **すべてオプトイン**: 革新的な操作はデフォルト OFF。設定で個別に有効化。
2. **フォールバック保証**: 新操作を有効にしても、従来の左クリック / 右クリック / D&D は必ず機能し続ける。
3. **段階的開示**: 初回は従来 UI、ユーザーが設定から「上級マウス操作」を有効化する導線。
4. **パフォーマンスファースト**: 視覚効果は `will-change` / `requestAnimationFrame` / GPU レイヤーを活用。60fps を下回る場合は自動的にエフェクトを無効化。
5. **発見性**: 有効化した機能は初回使用時にチュートリアルトースト（「セルを長押しすると…」）を表示。

---

*本提案は「マウスをもっと活かす」ことを主眼に、ソフトウェア・ハードウェア・異業種のインタラクションパターンを横断的に収集・翻訳したものです。全提案を一度に実装する必要はなく、ユーザーテストで効果の高いものから段階的に導入してください。*
