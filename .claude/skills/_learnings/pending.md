# 未処理の学び（Learnings Inbox）

このファイルは「スキル自己進化」の入口です。
セッション中に得た **スキルに反映すべき学び** をここに追記してください。
Stop フックがこのファイルを監視し、未処理エントリがあるとセッション終了前に
`skill-evolve` スキルによる取り込みを促します。

## 記録すべきもの

- スキルの記述と実際が食い違っていた（API変更、バージョン差異、誤情報）
- ハマった末に解決した非自明な問題（ビルドエラーの対処、クレートの落とし穴）
- 本家 CLaunch の挙動について新たに判明した仕様
- 効率の良い開発手順・検証手順を発見した

## 記録しなくてよいもの

- すぐ直せた単純なタイプミス・構文エラー
- このセッション限りの一時的な事情
- git 履歴やコードを見ればわかること

## エントリ形式

```markdown
## [YYYY-MM-DD] 短いスラッグ
- 対象スキル: iced-dev | clanch-spec | rlaunch-legacy | rlaunch-verify | (新規スキル名)
- 発見: 何がわかったか（1〜3行）
- 根拠: エラーメッセージ、URL、検証結果など
- 反映案: スキルのどこをどう直すか
```

※ 見出しの日付は実日付（数字）で書くこと。フックは `## [2026-07-04]` のような
実日付形式の行だけを未処理エントリとして数える（`[YYYY-MM-DD]` のままでは検知されない）。
