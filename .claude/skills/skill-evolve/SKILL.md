# skill-evolve — スキル自己進化

`.claude/skills/_learnings/pending.md` に溜まった学びを、対応するスキルファイルへ反映する。

## 手順

1. `.claude/skills/_learnings/pending.md` を読み、`---` 区切りより下の未処理エントリを列挙する。
   エントリが無ければ「反映すべき学びはない」と報告して終了。
2. 各エントリについて:
   a. `対象スキル` のスキルファイル（`SKILL.md` と参照ファイル）を読む。
   b. 学びの内容を **既存記述の修正 or 追記** として反映する。
      - 誤情報の訂正が最優先。古い記述は消し、正しい内容に置き換える（履歴はgitが持つ）。
      - ハマりどころは各スキルの「落とし穴 / Learnings」セクションに追記する。
      - 反映先が曖昧なら、最も再利用価値の高い場所を選ぶ。どのスキルにも属さない
        大きなテーマなら新スキルの新設を検討する（乱造はしない。目安: 既存スキルへの
        追記で足りるなら新設しない）。
   c. `description` が実態とずれたら frontmatter も更新する。
3. 処理したエントリを `pending.md` から削除し、`_learnings/archive.md` の末尾へ
   `処理日: YYYY-MM-DD` を付けて移動する（archive.md が無ければ作成）。
4. `pending.md` はテンプレートヘッダーと `---` 区切りだけの状態に戻す。
5. 反映結果を1行ずつ要約して報告する（どのスキルのどこを変えたか）。

## 品質基準

- スキルは **次のセッションの自分への指示書**。背景説明より「こうしろ/こうするな」を書く。
- 1つの学びは1箇所に書く。複数スキルに関係するなら本体1箇所＋他方からの参照。
- 反映後のスキルが古い記述と矛盾しないか、反映したファイル全体を読み直して確認する。
- 学びが「要検証」の状態（根拠が弱い）なら、断定で書かず `（要検証: ...）` を付ける。
