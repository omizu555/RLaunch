# RLaunch (iced版)

CLaunch 風ボタン型ランチャーの Rust ネイティブ実装。旧 Tauri + React 版の置き換え。
GUI は [iced](https://iced.rs) 0.14、Web 技術非依存の単一バイナリ。

## 旧版からの主な変更

- **単一バイナリ**（WebView2 / Node.js 不要）、**軽量**（常駐時ワーキングセット ~25MB。
  描画は tiny-skia のソフトウェアレンダリングで GPU メモリを使わない）
- **UIフォント設定**（設定 > 外観。既定は Yu Gothic UI、変更は再起動後に反映）
- 設定の「列数・行数」は**新規タブの既定値**に変更（旧版の全タブ一括リサイズは
  縮小時にアイテムが消える危険があるため廃止。既存タブはタブ右クリック→タブ設定で個別変更）
- カーソルアウト非表示は、**カーソルが一度ウィンドウに入ってから**「外に出たら消える」判定になる
  （表示位置=center 等でも表示直後に消えない）
- 旧版アイコンライブラリの **SVG アイコン**を `svg` ウィジェットで正しく表示（旧版は WebView 依存）。
  壊れた画像アイコンはデコード検証で安全にスキップ（絵文字にフォールバック、クラッシュしない）
- ウィンドウ透過は**透過テーマ（不透明度<1）を選んだときだけ**有効
  （不透明テーマではソフトウェア描画の残像を防ぐため透過を無効化）
- **ウィジェット機能は廃止**（旧データの WidgetItem は温存され「🧩（無効）」表示、解除のみ可能）
- 旧版で未配線だった機能を実装: 自動起動（HKCU Run）、autoHide 設定、
  アイテムの「実行時の大きさ」「管理者として実行」チェックの起動時反映
- CLaunch 風の「カーソルがウィンドウ外に出たら隠す」オプションを追加（設定 > 動作）
- アイコン抽出・lnk 解決を PowerShell 子プロセスから Win32 ネイティブ（SHGetFileInfoW /
  IShellLinkW COM）に置き換え

## データ互換

- 保存先は旧版と同一: `%APPDATA%\com.rlaunch.app\launcher-data.json`
  （**旧版のデータをそのまま読み込み、未知フィールドも保存時に温存**）
- テーマも旧形式の JSON（CSS 変数）を継続サポート: `%APPDATA%\com.rlaunch.app\themes\*.json`
  + バンドル25種（ビルトイン6 + サンプル19）
- 起動時に `launcher-data.bak.{1,2,3}.json` の3世代バックアップを自動作成
- ⚠ 旧 Tauri 版と**同時起動しない**こと（同じデータファイルを書き合うため）

## 操作（主要）

| 操作 | 機能 |
|---|---|
| `Ctrl+Space`（変更可） | 表示/非表示トグル |
| デスクトップをダブルクリック | 表示/非表示トグル |
| トレイ左クリック / 右クリック | トグル / メニュー（表示・設定・終了） |
| ファイルをグリッドへ D&D | アイテム登録（複数可、lnk 自動解決） |
| セルをドラッグ | 並べ替え（グループへドロップで格納） |
| 右クリック | コンテキストメニュー（起動/管理者/編集/解除/登録…） |
| 空セルをダブルクリック | ファイル選択して登録 |
| `Ctrl+F` | 検索（↑↓/Enter 起動、Ctrl+Enter でタブへ移動） |
| `Ctrl+ホイール` | セルサイズ変更 |
| タブバーでホイール | タブ切替 |
| ドラッグ中にタブへ 0.5 秒ホバー | タブまたぎ移動 |
| 矢印キー / Enter / Delete | セル選択 / 起動 / 解除 |
| 📌 | ピン留め（自動非表示を停止） |
| Esc | 閉じる（メニュー→ダイアログ→検索→ウィンドウの順） |

## ビルド

```powershell
cargo build --release   # 成果物: target/release/rlaunch.exe（単一ファイル）
```

開発時の検証手順は `.claude/skills/rlaunch-verify/SKILL.md` を参照。

## 構成

```
src/
├── main.rs            エントリ（単一インスタンス・イベントチャネル初期化）
├── app.rs             State / Message 定義・boot・購読
├── app_update.rs      update 実処理（操作ロジック）
├── app_windows.rs     ウィンドウ操作（表示/非表示・配置・ポップアップ）
├── app_registration.rs アイテム登録パイプライン
├── external.rs        外部イベント統合（ホットキー/トレイ/フック → Subscription）
├── tray.rs            システムトレイ
├── model/             データモデル（旧版 JSON 互換）・永続化・テーマ
├── platform/          Win32 層（起動/アイコン/lnk/モニター/フック/単一インスタンス）
└── ui/                ビュー（グリッド/タイトルバー/オーバーレイ/設定/ポップアップ）
```
